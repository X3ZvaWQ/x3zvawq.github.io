    • v2ray的WebSocket+TLS+Web

    自从19年6月4号左右遇到的搭了一个简单的ss然后整个ip被墙之后就对 ss 产生了阴影，再之后也一直用的付费机场的ss，然后这个机场似乎也被查封了，经历了好几天不能上外网的煎熬之后某次水群发现了一个有好多免费ss节点的一个txt文档，终于再次连上了外网访问到了google.com。

    经历了这么一个事件，深刻认识到了求人不如求己，别人梯子始终不会有自己的好，于是我还是选择了在自己的服务器上搭一个梯子，然而shadowsocks已经让我的服务器被墙了几个月了，所以这次要选择一个更加安全的方式惹，经过了一番查找最后目标锁定了v2ray的WebSocket+TLS+Web

    主要记录一下我踩过的坑，主要流程从简。

    • 系统 CentOS 7 (理论上其他的系统应该也可行)

    • 使用了lnmp一键安装包

    • v2ray 配置指南

    • 上述指南中主要参照部分（注意：该指南的配置文件中有坑）

    • 解决上面的坑的参考 (坑就是报400 Bad Request: too many Host headers)

    • 遇到的另一个坑的解决方案 (我并不是按这个方案解决的，详情看后文)
